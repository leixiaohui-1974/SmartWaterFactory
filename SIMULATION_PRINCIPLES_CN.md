# 水处理厂仿真模型核心原理

本文档详细解释了本项目中 `PlantSimulator`（水处理厂仿真器）所使用的核心物理和化学模型原理。

## 1. 核心设计思想

仿真器的设计目标是模拟一个动态系统，其水质参数会随着时间的推移和外部控制动作（如加药、曝气）的输入而发生变化。该模型包含了一些真实世界中常见的复杂性，如时间延迟和非线性效应，使其成为一个有价值的控制器测试平台。

---

## 2. 时间延迟（Time Delay）

### 原理
在真实的工业流程中，控制动作的效果并不是瞬时发生的。例如，从加药点到水质传感器所在位置，水需要流过一段管道，这个过程需要时间。这种现象被称为**时间延迟**或**传输延迟**。

### 实现方式
本仿真器使用了一个“先进先出”（FIFO）队列来模拟这个过程：
-   当控制器输出一个新的动作（如混凝剂剂量或曝气速率）时，该动作被放入一个模拟“管道”的队列的末端。
-   在每个仿真时间步，真正对水质产生影响的是从队列最前端取出的、最早进入“管道”的那个动作。
-   队列的长度由配置文件中的 `time_delay_steps` 参数决定，代表了延迟的时间步数。

---

## 3. 浊度（Turbidity）控制模型

### 原理
浊度是衡量水体浑浊程度的指标。在水处理中，通过投加**混凝剂**（Coagulant）来降低浊度。混凝剂使水中的悬浮颗粒物凝聚成更大的絮体，然后通过沉淀或过滤去除。

本模型基于一个简化的假设：浊度的降低速率与两个因素成正比：
1.  **当前的浊度水平**：水中悬浮物越多，可被去除的也越多。
2.  **混凝剂的投加剂量**：剂量越大，反应越快。

### 实现方式
浊度降低的计算公式如下：
```
浊度降低值 = 浊度衰减因子 * 混凝剂剂量 * 当前浊度
```
这是一个典型的一阶衰减模型，其中“浊度衰减因子” (`turbidity_decay_factor`) 是一个可在配置中调整的常数。

---

## 4. 溶解氧（Dissolved Oxygen, DO）动态平衡模型

### 原理
水中的溶解氧水平是一个动态平衡的结果，主要受两大过程影响：**增氧过程**和**耗氧过程**。

### 4.1. 增氧过程：曝气（Aeration）

#### 原理
曝气是向水中充入空气以增加溶解氧的过程。其效率遵循物理化学定律：
-   水的溶氧能力存在一个**饱和点** (`do_saturation`)，这取决于温度和压力。
-   增氧速率与**“缺氧程度”**（即饱和溶解氧与当前溶解氧的差值）成正比。水越缺氧，增氧越快。

#### 非线性效应
本模型还引入了**非线性效应**。在现实中，当溶解氧浓度非常接近饱和点时，再要提高一点点就会变得异常困难，即曝气效率会急剧下降。

#### 实现方式
为了模拟这一点，增氧速率的计算考虑了效率因子：
1.  计算“缺氧程度”：`缺氧差 = 饱和DO - 当前DO`
2.  根据缺氧程度计算效率惩罚因子：`效率因子 = (缺氧差 / 饱和DO) ^ (非线性系数 - 1)`
3.  计算有效增氧速率：`有效增氧速率 = 基础增氧速率 * 效率因子`
4.  最终增氧量：`增氧量 = 有效增氧速率 * 曝气速率 * 缺氧差`

其中，“非线性系数” (`aeration_non_linearity`) 可在配置中调整，该值越大，接近饱和点时的效率惩罚越严重。

### 4.2. 耗氧过程：自然消耗

#### 原理
水中的微生物活动或其他化学需氧物质会持续消耗溶解氧。在这个简化模型中，我们假设耗氧速率与当前水中的溶解氧浓度成正比。

#### 实现方式
耗氧量的计算公式如下：
```
耗氧量 = 基础耗氧速率 * 当前溶解氧
```
其中，“基础耗氧速率” (`do_consumption_rate`) 是一个可在配置中调整的常数。

---

## 总结

通过组合这些模型（时间延迟、浊度衰减、溶解氧动态平衡），`PlantSimulator` 提供了一个足够复杂且接近现实的虚拟环境，能够有效地测试和验证不同控制策略（如PID和开关控制）的性能。
