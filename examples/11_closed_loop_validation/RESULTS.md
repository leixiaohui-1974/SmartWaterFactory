# 扰动闭环验证报告

本报告记录了最新版 `run_closed_loop_demo.py` 的运行情况，并说明如何理解生成的结果。

## 1. 执行指令

在项目根目录运行：

```bash
python -m examples.11_closed_loop_validation.run_closed_loop_demo
python visualize_log.py --log-file data/closed_loop_validation.csv --output-image data/closed_loop_validation.png
```

- 第一个命令会生成 `data/closed_loop_validation.csv`，包含时间戳、浊度/溶解氧、控制器输出以及扰动因子。  
- 第二个命令会渲染三行子图 `data/closed_loop_validation.png`，直观展示系统输入输出与扰动之间的关系。  
- 由于 `visualize_log.py` 预先配置了 `SimHei/Microsoft YaHei/DejaVu Sans` 字体，本次绘图未出现中文字体缺失警告。

## 2. 关键指标（来自 CSV）

- 模拟步数：**200**（每步 1 分钟，共 3 小时 20 分钟）。  
- 浊度：25.00 NTU → **3.22 NTU**（设定值 5.00 NTU，残余误差 1.78 NTU）。  
- 溶解氧：3.92 mg/L → **8.25 mg/L**（设定值 8.50 mg/L，残余误差 0.25 mg/L）。  
- 浊度首次跌破 6 NTU 的时间：第 23 步；在第二次扰动（110–124 步）后约 30 分钟重新回到设定区附近。  
- 溶解氧在首分钟即跨过 8 mg/L，随后始终保持在 ±0.3 mg/L 范围内。  
- 稳态控制输出（最后 5 个采样点）：混凝剂投加量约 **0.46**，曝气速率约 **15.30**。

扰动设定：

- 基础扰动为振幅 0.3 NTU 的慢变化正弦波，模拟原水浊度缓慢波动。  
- 脉冲 1（第 40–54 步）：+2.0 NTU，模拟暴雨或进水尖峰。  
- 脉冲 2（第 110–124 步）：+1.0 NTU，模拟短时负荷增加。

## 3. 图像解读

打开 `data/closed_loop_validation.png`，可见：

1. **顶部：浊度控制**  
   - 蓝色实线表示出水浊度，蓝色虚线为设定值 5 NTU。  
   - 两个扰动脉冲在曲线上形成明显峰值。PID 控制器提升药剂投加后，浊度在几十分钟内回落至设定区附近。

2. **中部：溶解氧控制**  
   - 红色实线与虚线几乎重合，说明曝气 PID 即使在浊度扰动期间也能稳定提供充足溶解氧。  
   - 绿色文字框给出均值和标准差，可用于评估波动程度。

3. **底部：控制输入与扰动因子**  
   - 橙色曲线：混凝剂投加量；绿色曲线：曝气速率；紫色虚线：浊度扰动因子。  
   - 当紫色曲线出现正脉冲时，橙色曲线随之升高，验证控制器根据扰动自动加药；曝气速率则相对稳定。

## 4. 结果评估

- 外部扰动使浊度长期低于设定值，但误差（1.78 NTU）仍在脚本设定的 ±2.0 NTU 容差内，满足更严格的出水标准。  
- 溶解氧控制保持在 ±0.3 mg/L 的目标范围内，无明显超调或振荡。  
- 通过统一的图像可以快速对照「扰动 → 控制输出 → 过程变量」链路，便于定位控制策略是否足以抵消扰动。

## 5. 继续探索

- 若要测试不同扰动，可直接修改 `_turbidity_disturbance` 中的正弦/脉冲参数，再运行上述指令。  
- 如需观察更激烈的场景，可增大脉冲幅度或频率，同时适当放宽容差或重新调节 PID 参数。  
- 若要记录其他噪声（例如溶解氧传感器偏差），可在脚本中添加新的扰动列，并在 `visualize_log.py` 中扩展相应绘图逻辑。

通过以上步骤，可以全面理解扰动的类型、控制器的响应以及最终的水质走势，方便进一步优化智能水厂控制策略。
