# 示例 18：HIL 演示链路

本示例展示如何使用仓库中已经实现的硬件在环（HIL）能力，覆盖三条调用路径：

- 命令行脚本：快速本地演示
- REST API：面向服务集成
- MCP 工具：面向智能体 / 协议调用
- 静态前端面板：零构建方式连接 REST 接口，带自动轮询、自动步进与趋势图

适用场景：

- 教学演示 AquaMind HIL 基础能力
- 验证场景切换、控制下发、故障注入
- 为后续前端对接 `docs/claude/AquaMind_HIL_Simulator.jsx` 做接口预演

## 1. 命令行演示

最小演示：

```bash
python scripts/run_hil_demo.py --steps 10 --scenario steady --output outputs/hil_demo_summary.json
```

带故障注入的演示：

```bash
python scripts/run_hil_demo.py \
  --steps 10 \
  --scenario turbidity_spike \
  --coagulant-dose 6 \
  --aeration-rate 10 \
  --fault-sensor turbidity \
  --fault-mode stuck \
  --fault-value 42 \
  --output outputs/hil_demo_fault.json
```

输出文件中包含：

- `latest_snapshot`：最后一步快照
- `snapshots`：完整 HIL 时序
- `controls`：执行器命令
- `fault`：故障注入配置

## 2. REST API 演示

也可以直接运行一键端到端脚本：

```bash
python scripts/run_hil_rest_e2e.py --output outputs/hil_rest_e2e_summary.json
```

该脚本会自动完成：

1. 启动 API 服务（默认关闭 WebSocket，方便本地演示）
2. 登录并获取 Token
3. 启动 HIL 会话
4. 下发控制量
5. 步进仿真
6. 注入故障并再次步进
7. 导出最终状态到 JSON

如果已经启动 API 服务，可以通过如下接口操作 HIL：

### 2.1 获取可用场景

```bash
curl -X GET http://127.0.0.1:5000/api/hil/scenarios \
  -H "Authorization: Bearer <TOKEN>"
```

### 2.2 启动 HIL 会话

```bash
curl -X POST http://127.0.0.1:5000/api/hil/start \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d "{\"scenario\": \"steady\", \"random_seed\": 7, \"dt_s\": 1.0}"
```

### 2.3 下发控制量

```bash
curl -X POST http://127.0.0.1:5000/api/hil/<SIM_ID>/control \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d "{\"coagulant_dose\": 6.0, \"aeration_rate_ma\": 12.0}"
```

### 2.4 步进仿真

```bash
curl -X POST http://127.0.0.1:5000/api/hil/<SIM_ID>/step \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d "{\"steps\": 5}"
```

### 2.5 注入传感器故障

```bash
curl -X POST http://127.0.0.1:5000/api/hil/<SIM_ID>/fault \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d "{\"sensor_name\": \"turbidity\", \"mode\": \"stuck\", \"value\": 42.0}"
```

### 2.6 查询状态

```bash
curl -X GET http://127.0.0.1:5000/api/hil/<SIM_ID>/status \
  -H "Authorization: Bearer <TOKEN>"
```

## 2.7 静态前端面板

```bash
cd examples/18_hil_demo
```

```text
http://127.0.0.1:8088/rest_panel.html
```
